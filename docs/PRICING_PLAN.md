# 定价与权益方案（按月订阅 + 积分制）

目标：把“按次收费”调整为“按月付费 + 积分（Credit）消耗”，覆盖 Nano Banana 成本且保持起步价不高；同时支持「注册后每日赠送 1 积分」。

## 1. 成本与定价约束（不亏钱的底线）

### 1.1 单次调用成本（你的数据）
- Nano Banana 标准版：$0.025 / 张
- Banana Pro：$0.05 / 张

### 1.2 站内积分消耗建议（按成本比例定价）
- 1K（走 Nano Banana 标准版模型）：1 积分 / 张
- 2K（走 Banana Pro 模型）：4 积分 / 张
- 4K（走 Banana Pro 模型）：5 积分 / 张

说明：这里的 “Pro” 分为两层含义：
- Pro 会员：你网站的订阅档位（权益、每月赠送积分不同）
- Banana Pro：第三方图改图模型（成本更高：$0.05/张）

因此我们可以让 Pro 会员默认仍走 1K（标准模型，成本 $0.025/张，1 积分/张），只有用户主动选择 2K/4K 才走 Banana Pro 模型并消耗更多积分。

### 1.3 亏损风险来源（需要预留空间）
- 支付通道手续费（Stripe 一般 3%～5%）
- 生成结果存储/带宽（Vercel Blob 或本地存储在生产会转为对象存储）
- 客服与坏账（退款/拒付）

因此“套餐内积分”要按“用户全部用完也不亏”的口径来设计。

## 1. 推荐方案（主推：¥9.9 起步 + 会员可用 Pro）

说明：本文中的
- 分辨率/画质：1K / 2K / 4K（影响调用成本与积分消耗）
- 图片尺寸：输出图片的长宽比（如 1:1、3:4、9:16 等）

### 2.1 免费版（注册用户）
- 价格：¥0 / 月

- 加购积分：不可用（必须先成为会员）
- 图片尺寸：固定 1:1，不可更改长宽比

备注：为避免“免费日赠 + 会员月赠”叠加导致亏损，成为会员后不再领取每日赠送，改为按月发放会员积分。

- 分辨率（画质）：支持 1K / 2K / 4K（默认 1K）
- 图片尺寸（长宽比）：支持 1:1 / 3:4 / 9:16 等
- 加购积分：可用（会员专享）

换算示例：

### 3.2 使用与有效期建议（避免争议，保持简单）

## 5. 备选方案（如果你希望会员也保留每日赠送）

不推荐（会显著抬高亏损风险），但如果一定要做，建议把“每日赠送”改为：
- 仅免费用户可领
- 或者改为每周 2 积分（当周有效）

## 6. 为什么这个方案不容易亏钱（直觉版结论）
- Pro 按分辨率阶梯消耗：默认 2 分，2K/4K 更高消耗控量
- ¥9.9 起步的套餐内积分更保守：即使全用完也更容易覆盖成本与手续费
- 会员专享加购：把高消耗用户引导到“额外付费”，避免被少数重度用户拖垮
